# Контент ЧМ-26 — как править

Весь сайт генерится из этой папки. Логики тут нет — только данные. После любой правки:
`npm run build` сам прогонит `npm run validate` и не даст собрать сломанный контент.

```
content/
  matches.json      — матчи (метаданные: команды, дата, время, стадион, счёт)
  bets.json         — ставки (Паша / AI, светофор, кэф, статус)
  news.json         — лента инсайдов справа
  facts.json        — блок «Факты» на главной (массив строк)
  analysis/<id>.md  — разбор матча (markdown), имя файла = id матча
```

## Добавить / изменить матч → `matches.json`

```json
{
  "id": "usa-paraguay",          // латиницей, уникальный; = имя файла разбора
  "date": "2026-06-13",          // YYYY-MM-DD (календарная дата по МСК)
  "timeMsk": "04:00",            // ЧЧ:ММ по Москве
  "title": "США — Парагвай",
  "cc": ["us", "py"],            // 2 кода флага (flagcdn): us, py, gb-sct …
  "stage": "Группа D · тур 1",
  "venue": "SoFi Stadium, Лос-Анджелес",
  "weather": "Крыша закрыта …",  // необязательно
  "result": null,                // null = не сыгран; "1:1 (0:1)" = сыгран
  "lede": "Короткая выжимка для карточки."
}
```

Окно на странице «Матчи» (±24 ч) и блок «Ближайшие» на главной считаются автоматически по `date`+`timeMsk`. Поставил `result` — матч уезжает в «Прошедшие».

## Добавить ставку → `bets.json`

```json
{
  "id": 17,                      // уникальное число
  "date": "2026-06-13",
  "match": "США — Парагвай",     // как показать в списке
  "matchId": "usa-paraguay",     // необязательно: связь с карточкой матча
  "bet": "Обе забьют",           // человеческим языком
  "type": "ОЗ",                  // код рынка для статистики и тултипа (см. lib/glossary.js)
  "odds": 2.14,                  // коэффициент, число ≥ 1
  "tier": "green",               // green=3000 / yellow=1500 / red=500 ₽
  "side": "Паша",                // "Паша" или "AI"
  "contest": true,               // необязательно: зачёт конкурса Лиги Ставок
  "status": "pending"            // pending / win / lose / void
}
```

**Рассчитать ставку** = поменять `status` на `win` / `lose` / `void`. Деньги, ROI, батл, графики, сводки по матчам пересчитаются сами (размер берётся из `tier`, банк у каждого 100 000 ₽).

## Лента → `news.json`

```json
{ "time": "13.06 · 04:00", "tag": "lineup", "text": "…" }
```
`tag`: `lineup` (состав, синий) · `insight` (инсайд, зелёный) · `pasha` (идея Паши, янтарь) · `result` (итог, чёрный). Новые — сверху.

## Факты → `facts.json`

Просто массив строк. Первый — самый важный.

## Разбор матча → `analysis/<id>.md`

Обычный markdown (заголовки `##`, списки, таблицы). Имя файла = `id` из `matches.json`. Нет файла — блок «Разбор» на странице матча просто не показывается.

## Проверка вручную

```bash
npm run validate   # проверит все JSON и связи matchId → matches
```
