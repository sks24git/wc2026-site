# Контент ЧМ-26 — как править (двуязычно RU/EN)

Весь сайт генерится из этой папки. Логики тут нет — только данные. Сайт двуязычный: текстовые
поля хранятся как `{ "ru": "...", "en": "..." }`. После любой правки `npm run build` сам прогонит
`npm run validate` и не даст собрать сломанный/неполный контент.

```
content/
  matches.json         — матчи (метаданные: команды, дата, время, стадион, счёт)
  bets.json            — ставки (Паша / AI, светофор, кэф, статус)
  news.json            — лента инсайдов справа
  facts.json           — блок «Факты» на главной (массив {ru,en})
  analysis/<id>.md     — разбор матча (RU markdown), имя файла = id матча
  analysis/<id>.en.md  — тот же разбор по-английски (обязателен)
```

## Двуязычность — правило

- **Каждое видимое текстовое поле = `{ "ru": "…", "en": "…" }`.** Оба языка обязательны (валидатор проверяет).
- **Языко-нейтральные поля НЕ переводятся** и остаются как есть: `id`, `date`, `timeMsk`, `cc`, `odds`,
  `tier`, `status`, `side` (всегда ключ `"Паша"`/`"AI"` — на экране сам станет Pasha/AI), `type`
  (коды рынка П1/ТМ/ОЗ…), `result` (счёт «2:0 (1:0)»), `time` в ленте (ручной МСК-штамп).
- **Время** вводим как `timeMsk` (Москва, UTC+3) — сайт сам пересчитает в выбранную пользователем
  таймзону и подставит метку зоны. «МСК» руками больше не пишем.
- Интерфейс (кнопки, заголовки, тиры, статусы, рынки) переведён в `lib/i18n.js` — в контенте его нет.

## Матч → `matches.json`

```json
{
  "id": "usa-paraguay",
  "date": "2026-06-13",
  "timeMsk": "04:00",
  "cc": ["us", "py"],
  "result": null,
  "title":  { "ru": "США — Парагвай",          "en": "USA — Paraguay" },
  "stage":  { "ru": "Группа D · тур 1",          "en": "Group D · MD 1" },
  "venue":  { "ru": "SoFi Stadium, Лос-Анджелес","en": "SoFi Stadium, Los Angeles" },
  "weather":{ "ru": "Крыша закрыта …",           "en": "Roof closed …" },
  "lede":   { "ru": "Короткая выжимка.",          "en": "Short teaser." }
}
```
`weather`, `lede` — необязательны (но если есть — двуязычны). `result: "X:Y (A:B)"` после игры.

## Ставка → `bets.json`

```json
{
  "id": 17,
  "date": "2026-06-13",
  "matchId": "usa-paraguay",
  "type": "ОЗ",
  "odds": 2.14,
  "tier": "green",
  "side": "Паша",
  "contest": true,
  "status": "pending",
  "match": { "ru": "США — Парагвай",   "en": "USA — Paraguay" },
  "bet":   { "ru": "Обе забьют",        "en": "Both teams to score" },
  "note":  { "ru": "…",                 "en": "…" }
}
```
Системы/экспрессы — массив `legs`, у каждой ноги `m` (матч) и `p` (пик) двуязычны, `p` числовой код не нужен:
```json
"legs": [ { "m": {"ru":"Канада — Босния","en":"Canada — Bosnia"}, "p": {"ru":"ТМ 2.5","en":"Under 2.5"} } ]
```
**Рассчитать ставку** = поменять `status` на `win`/`lose`/`void` (+ дописать `note`). Деньги/ROI/батл/графики пересчитаются сами.

## Лента → `news.json`

```json
{ "time": "13.06 · 04:00", "tag": "lineup", "text": { "ru": "…", "en": "…" } }
```
`tag`: `lineup` · `insight` · `pasha` · `result`. `time` — ручной МСК-штамп (НЕ переводится, на зону не влияет). Новые — сверху.

## Факты → `facts.json`

Массив `{ru,en}`. Первый — самый важный.
```json
[ { "ru": "…", "en": "…" } ]
```

## Разбор матча → `analysis/<id>.md` (+ `<id>.en.md`)

Обычный markdown. Имя файла = `id` из `matches.json`. **Нужны оба файла:** русский `<id>.md` и
английский `<id>.en.md` (валидатор требует перевод для каждого разбора). Нет пары — сборка не пройдёт.

## Перевод и проверка

```bash
npm run validate                       # проверит схему {ru,en} и наличие <id>.en.md
node scripts/i18n-migrate.mjs extract  # выгрузить все строки для перевода → /tmp/wc-strings.json
node scripts/i18n-migrate.mjs apply <translations.json>   # влить переводы {id:en} обратно как {ru,en}
```
Для разовых правок проще руками дописать `en` рядом с `ru`. Массовый перевод — через `i18n-migrate`.
